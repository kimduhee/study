### A 시스템 => B 시스템 호출하여 B 시스템에서 기능 처리 후 처리상태를 A시스템에 전달해 줘야함 
+ B 시스템의 처리시간이 픽스되지 않음
+ A 시스템이 호출시 타임 아웃 발생 소지 있음
+ A 시스템은 B 시스템의 처리 상태 알아야함
<pre><code>해결
1.A시스템이 B시스템에 요청
2.B시스템은 '처리요청'상태 값 A시스템에 전달하고 요청처리(이벤트 및 스레드로 처리)
3.A시스템은 전달받은 '처리요청'상태 값 저장
4.B시스템에서는 처리 후 A시스템에 처리상태값(처리완료 or 에러)전달
5.A시스템은 B시스템에게 전달받은 처리상태값 저장 및 후처리 작업 진행
</code></pre>

+ DB 구조는 유지
+ 이전 프로세스 유지(신규 프로세스 오류 대비 롤백)
+ 일부 인원만 신규 프로세스 테스트
+ 화면 등 프로세스 변경
<pre><code>해결
1.Feature Flag 방식적용
2.테스터 사용자 정보에 테스트 업무 ID 부여
(로그인 변경 테스터 'login_tester'로 부여)
3.화면이나 서버에서 'login_tester'로 분기처리
(테스터만 버튼 보임, 신규 url이동 등)
4.각 기능별 ID부여하여 담당하는 기능들만 테스트
</code></pre>
